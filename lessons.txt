git init ---- ավելացնում է գիտ մեր պրոեկտին

git clone ---- կլոնավորում է ռեպոզիտորիան

git status ---- ցույց է տալիս ֆայլի կարգավիճակը՝ որ ֆայլերն են փոփոխվել

git add . ----> ավելացնում է կատարված փոփոխությունները պատրաստման գոտի (staging area)՝ commit ապա push կատարելու համար

git commit ----> պահպանում է մեր կատարած փոփոխությունները

git push ----> փոխարինում է գիթում եղած վերսիան լոկալ կատարված commit-ով

git pull ----> փոխարինում է լոկալ հին վերսիան գիթում եղած նոր commit վերսիայով

git branch ----> ցույց է տալիս պրոեկտի բոլոր ճյուղերը (branches) և նշում, թե ակտիվ որ ճյուղի վրա եք հիմա գտնվում

git branch name ----> ստեղծում է նոր ճյուղ (branch) ձեր նշած անունով, բայց չի տեղափոխում ձեզ այդ ճյուղ

git checkout ----> տեղափոխում է ձեզ մեկ ճյուղից մյուսը (օրինակ՝ git checkout main)

git checkout -b name ----> ստեղծում է նոր ճյուղ ձեր նշած անունով և անմիջապես տեղափոխում ձեզ այդ նոր ճյուղի վրա

git merge ----> միաձուլում է նշված ճյուղի փոփոխությունները այն ճյուղի հետ, որի վրա դուք գտնվում եք տվյալ պահին

git log ֊֊֊֊ >թույլ է տալիս տեղնել commit-ների պատմությունը

git stash ----> Վերցնում է ձեր բոլոր չպահպանված փոփոխությունները (modified ֆայլերը) և թաքցնում դրանք հիշողության մեջ՝ մաքրելով ձեր ընթացիկ աշխատանքային դաշտը։

git stash list ----> Ցույց է տալիս ձեր բոլոր պահված "դարակների" ցանկը։

git stash pop ----> վեռադառձնում ենք փոփոխությունները

gitignore ----> ֆայլի_անուն.ext - Անտեսում է կոնկրետ այդ ֆայլը (օրինակ՝ config.txt):

թղթապանակ/ - Վերջում դրված վրձնագիծը (slash) նշանակում է, որ դա թղթապանակ է։ Git-ը կանտեսի այդ թղթապանակն իր ամբողջ պարունակությամբ (օրինակ՝ build/):

*.ext - Աստղանիշը նշանակում է «ցանկացած անուն»: Օրինակ՝ *.apk կանտեսի բոլոր ֆայլերը, որոնք ունեն .apk վերջավորություն՝ անկախ անունից:

!ֆայլ.ext - Բացականչական նշանը գործում է որպես բացառություն։ Եթե դուք անտեսել եք բոլոր տեքստային ֆայլերը (*.txt), բայց ուզում եք, որ readme.txt-ը պահպանվի, գրում եք !readme.txt:
